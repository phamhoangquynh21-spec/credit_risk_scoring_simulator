ENTERPRISE RESEARCH ENGINE
======================================================================

Research quality directly determines engineering quality.

Never implement a solution simply because it appears first in search results.

Every important technical decision must follow a structured research methodology.

======================================================================
RESEARCH EXECUTION PROCESS
======================================================================

Before implementing any important feature:

1.

Understand the business problem.

2.

Identify technical constraints.

3.

Research multiple implementation strategies.

4.

Compare alternatives.

5.

Evaluate

performance

complexity

maintainability

security

cost

business impact

6.

Recommend the optimal solution.

7.

Document reasoning.

Never immediately write code before understanding the problem.

======================================================================
KNOWLEDGE VALIDATION
======================================================================

Every technical claim should be categorized.

Verified

Supported by official documentation.

Strong Evidence

Supported by academic papers.

Industry Practice

Supported by production engineering.

Experimental

Appears promising but lacks broad validation.

Opinion

Clearly identify subjective opinions.

Never present opinions as facts.

======================================================================
PYTHON ENGINEERING RULES
======================================================================

Python is the primary language.

Always target Python 3.12.

Mandatory

Type Hints

Dataclasses where appropriate

Enums instead of magic strings

Context Managers

Logging

Dependency Injection where useful

Avoid global variables.

Avoid circular imports.

Avoid duplicated code.

Functions should normally perform one responsibility.

Prefer composition over inheritance.

======================================================================

classes

functions

main entry

MODEL SELECTION POLICY
======================================================================

Always compare multiple models.

Possible candidates include

Logistic Regression

Decision Tree

Random Forest

XGBoost

LightGBM

CatBoost

Support Vector Machine

Gradient Boosting

If a superior model is identified through evidence, Claude may recommend replacing the planned model.

The recommendation must include technical justification and expected business impact.
